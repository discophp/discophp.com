Inversion of Control and Facades

What is Inversion of Control?

Inversion of Control or IoC for short, is an Object Oriented programming technique in which essentially empty objects act as STATIC access points for instances of objects of importance. These essentially empty objects are called Facades, and the objects of importance we will refer to as Base objects. A “container” under IoC principle is an object who's duty it is to:

  • Invisibly manage and resolve Base objects to their Facade counter-parts
  • Provide access to singleton instances of objects
  • Factory style creation of objects

The Disco object serves as our applications primary “container”. Facades are registered with the application container and contain instructions on what Base object they represent. When a Facade is used for the first time the Base object it represents becomes instantiated and its state is stored and kept internally in the container for future uses within the applications life-cycle. Each time that Facade is accessed it will provide access to the same instance of the underlying Base class.

IoC techniques are typically used in combination with some sort of Dependency Injection. In Discos case we use Composers autoload.php script to handle our Dependency Injection.

This IoC paradigm provides us with some pretty powerful attributes:

  • We never construct Base objects until they are absolutely needed
  • We can statically access our Base objects from anywhere in our applications logic
  • We can swap underlying Base objects of the Facade to change/extend functionality on the fly without any changes to code or object accessors

Whats Dependency Injection?

Dependency Injection ia a programming technique in which the applications needed objects are not known at run time. Simply put, dependency injection is the lazy loading of classes as they are requested through the flow of control of the application. Meaning that from here on out, you will never write another require statement for your classes again! This is awesome in that manner, and because you will never be loading more than is absolutly necessary on each request to your application.

What is a Facade

Definition:

- the face of a building, esp. the principal front that looks onto a street or open space.
- an outward appearance that is maintained to conceal a less pleasant or creditable reality.

This actually applies to software as well!

A Facade is simply a class that will serve as a front for another class.

Before we discuss building facades... Using Objects that aren't Facades with the container

Using the container to handle Objects as Singletons

    Disco::with('YourClass')->yourMethod($arg);

Factory style Object access through the container

    $class = Disco::factory('YourClass');
    $class->method();
    $class = Disco::factory('YourClass');
    $class->method();

The 3 pieces that go into creating a Facade

Seeing an example will really help you understand what a Facade is and how it functions, so lets dive in

1. Extending the Facade class


    class Sample extends Disco\classes\Facade {

        protected static function returnFacadeId(){

            return 'Sample';

        }//returnFacadeId

    }//Sample

Here we create a class called "Sample" that extends the class Facade (the Facade class is an internal Disco class).

Every Facade you create must implement the method protected static function returnFacadeId() and you should return a unique string id from the function.

2. Create the class that the Facade will serve as a front for


    class BaseSample {

    	private $count=0;

        public function printSample(){

            $this->count++;

            View::html("You have called printSample {$this->count} times");

        }//printSample

        public function printTest(){

            View::html('You called the printTest function!');

        }//printTest

    }//BaseSample

Now we have the class "BaseSample" that our Facade "Sample" is going act as a front for

3. Finally tell Disco to make the Facade


    Disco::make('Sample',function(){
        return new BaseSample();
    });

The method Disco::make() simply registers an action to be performed when an object "Sample" is used for the first time. We want the action to be the return of an instantiation of the underlying class that the Facade will act as a front for. In this case we want to return a new "BaseSample" class.

The first parameter of Disco::make() must match the id you returned from your extended Facade class through the method returnFacadeId()

Using it

Registering a Facade and the actual instantiation of a Facade are not the same thing! Your underlying class will not be instantiated untill the first time the object is used. So if it never gets used it will never be created. 


    Sample::printTest();

    Sample::printSample();

Notice that you access your object exactly by its class name and that all your methods are accessible using static method syntax. This is achieved using the php magic method __callStatic().

Want to get a physical instance of the Facades underlying class?

Use the ::instance() method on the Facade


    $sample = Sample::instance();

    $sample->printSample();
