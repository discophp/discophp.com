Session Facade

The Session Facade provides an easy wrapper around using $_SESSION data

The __construct() method of the Session Facade provides some important functionality:

First it will check to see if the session has already seen a request and if not store the IP address of the user in the session variable $_SESSION['IP_REFER_CHECK']

On subsequent requests the IP address will be double checked to make sure the session hasn't been hijacked. If its found to differ the application is going to redirect the user to a /404 page.

For security reasons when we load up the application some php_ini variables are going to be changed

    ini_set('session.use_trans_sid',0);
    
    ini_set('session.use_only_cookies',1);

Basic usage

Seeing if a variable is set


    if(Session::has('user_id'))
        View::html('we have a user ID');
    else 
        View::html('no user');

Getting a set variable

    View::html('logged in user has id: '.Session::get('user_id'));

Setting a variable

    Session::set('user_id',500);

Removing a variable

    Session::remove('user_id');

Regenerating a session ID

    Session::regen();
It is recommended that you call Session::regen() after each login of a user

Reset a session

    Session::reset();

Flush/Clear/Wipe a session

    Session::flush();
