Request LifeCycle

A crucial aspect of application development is understanding how things are working under the hood.

Here we will detail each piece of the applications components in a step-wise process.

First off, all requests in Disco are routed through index.php, unless there is a filename given with a matching file on the system to execute the request

If your using Apache2 as a Web Server you must enable modrewrite.

sudo a2enmod rewrite

The first line in index.php is going to require the core Disco file core/Disco.core.php

The core Disco file will then take care of setting up the environment configuration using the file core/Prep.core.php

It will then register the default Facades with Disco

Now our flow of control is back to index.php

Now any user set required files are included

Event listeners are registered

Routers begin being processed to resolve a path with an action

Finally View::printPage(); is called and the View is printed
