Request Life-Cycle

Here we will detail each piece of the applications request life-cycle in a step-wise process.

First off, all requests in Disco are routed through public/index.php, unless there is a filename given with a matching file on the system to execute the request. See the .htaccess file and check out the modrewrite rules.

You must enable modrewrite if your using the Apache2 Server.

sudo a2enmod rewrite

  • The first line in public/index.php is going to require the core Disco file core/Disco.core.php
  • The core Disco file will then take care of setting up the environment configuration using the file core/Prep.core.php
  • Disco will then register the default Facades with its container by calling the method Disco::registerDefaults()
  • Now our flow of control is back to public/index.php
  • All user written logic begins being processed
  • Routers begin being processed to resolve a endpoint url with an action by means of a Closure or Controller
  • If no Router matched the requested URL then redirect the user to a /404 page
  • Finally View::printPage(); is called and the View is printed
